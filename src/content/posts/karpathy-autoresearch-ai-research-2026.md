上个月 Karpathy 发了个新项目，叫 autoresearch。GitHub 上已经 87000 多个 star 了。

项目本身很小，核心代码就三个文件。但它的思路挺颠覆的：你给 AI agent 一个真实的 LLM 训练环境，告诉它"去跑实验吧"，然后你就去睡觉了。第二天早上起来，看它的实验日志——如果运气好的话，模型比你手动调参的结果更好。

## 这东西到底怎么跑的

autoresearch 的架构简单到让人意外。

`prepare.py` 负责数据预处理和 tokenizer 训练，这个文件不许改。`train.py` 包含完整的 GPT 模型、优化器和训练循环，这个文件由 AI agent 来改。`program.md` 是给 agent 的指令，这个文件由人来写。

每次实验的训练时间固定 5 分钟（不算启动和编译），用验证集的 bits per byte 作为指标——越低越好。agent 改完 `train.py`，跑一次训练，看指标有没有提升，提升了就保留，没提升就回滚，然后再来一轮。

Karpathy 说，核心变化在于：你不再直接写 Python 代码了。你在写 `program.md`——一个 Markdown 文件，用来定义你的"研究组织"该怎么运作。你编程的对象从代码变成了指令。

他在推特上写了一段挺有意思的话：

> 以前，前沿 AI 研究是肉做的计算机（指人类）在吃饭、睡觉、社交的间隙完成的，偶尔用声波互联（指开会）同步一下。那个时代已经过去了。研究现在完全是 AI agent 自主群体在天空中的计算集群超级结构上运行的领域。agent 说它们现在是代码库的第 10205 代了，不过没人能确认这是不是真的，因为"代码"已经变成了一个超出人类理解的自修改二进制。

这段话有 Karpathy 一贯的风格——半认真半开玩笑，但指向的东西是真实的。

## 为什么这件事值得关注

说实话，"AI 辅助科研"这个说法已经喊了好几年了。AlphaFold 预测蛋白质结构，GPT 帮写论文摘要，Copilot 帮写代码——这些都是辅助。人类做决策，AI 做执行。

autoresearch 不太一样。它把决策权也交出去了。

agent 自己决定改什么超参数，自己决定换什么架构，自己评估结果，自己决定保留还是回滚。人类退到了更高一层——你不再是做实验的人，你是设计"做实验的方式"的人。

这让我想到一个类比。以前写代码是直接操作机器指令，后来有了编译器，你写高级语言，编译器帮你翻译。再后来有了框架，你写配置文件，框架帮你搭基础设施。现在 Karpathy 说的其实是同一件事的下一个版本：你写 Markdown，AI 帮你做研究。

当然，这个类比有它的局限。编译器翻译代码是确定性的，AI 做研究不是。它可能跑偏，可能在错误的方向上浪费算力，可能优化了一个不重要的指标而忽略了真正的问题。Karpathy 自己也承认 `program.md` 的设计需要迭代——你得找到那个能让 agent 最高效地做研究的"研究组织代码"。

## 实际效果怎么样

从 autoresearch 的 `progress.png`（一个记录实验进展的图表）来看，agent 确实在持续改进模型。不是那种"随机波动然后碰巧变好"的改进，是有趋势的、持续的下降。

但这里有个重要的前提：这个项目的实验规模很小。单 GPU，5 分钟训练，一个简化版的模型。在真实的研究场景中，一次实验可能要跑几天甚至几周，涉及分布式训练、大规模数据集、复杂的评估流程。autoresearch 目前离那个级别还很远。

不过 Karpathy 的野心显然不止于此。他在 README 里暗示了方向：加更多 agent，找更好的 `program.md` 设计，让整个"研究组织"自动扩展。他说这个项目的默认 `program.md` 故意做得很基础，就是为了让社区去迭代。

## 我的几个想法

第一，这件事最有意思的地方不是技术本身，而是它改变了人和研究的关系。以前做 ML 研究需要你同时是程序员、数学家和实验设计师。现在你可能只需要是实验设计师——你得知道什么实验值得做，但不需要知道怎么把每一行代码写出来。门槛降低了，但判断力变得更重要了。

第二，agent 做研究的"可复现性"问题会变得很棘手。传统研究讲究可复现——你按论文里的方法做，应该能得到同样的结果。但如果研究过程本身包含了 agent 的随机决策呢？你复现的到底是代码，还是那个 agent 在那个特定时间点的决策路径？

第三，Karpathy 这个项目最聪明的地方是它的定位。他没有说"AI 取代研究员"，他说的是"你编程的对象从代码变成了指令"。这个框架让人更容易接受——你还是在编程，只是编程的抽象层次变了。比起那些"AGI 即将取代一切"的叙事，这种渐进式的描述反而更有说服力。

autoresearch 的 star 数还在涨。87000 个 star 意味着很多人在关注这个方向。不知道有多少人真的在用它跑实验，但至少说明了一件事：大家开始认真考虑"让 AI 自己做研究"这件事了。

这到底是好事还是坏事，我现在说不好。但它确实在发生。
