7 月这一个月，AI Agent 领域发生的事情比过去半年加起来还多。

MCP 1.0 正式发布了。AutoGen 0.9 带着动态协商机制来了。Accenture 说自己部署了 2500 多个 Agent，摩根大通上线了覆盖 200+ 流程的 Agent 平台。Gartner 说这个市场今年 47 亿美金，每年翻 1.68 倍。

数据挺漂亮，新闻稿也一篇接一篇。但我更想说点不一样的东西。

## MCP 1.0：Agent 之间的"普通话"

7 月 1 日，Anthropic 联合 OpenAI、Google、Meta、Microsoft 等 20 多家公司发布了 MCP（Model Context Protocol）1.0 规范。

MCP 是什么？简单说，就是给 AI Agent 之间定的一套通信协议。你定义一个工具接口，任何兼容 MCP 的 Agent 都能调它，不用为每个框架写适配器。

这件事的意义被很多人低估了。

2024 年和 2025 上半年，Agent 圈是什么局面？每个框架都有自己的工具调用方式。LangChain 的 Tool、OpenAI 的 function calling、AutoGen 的 tool schema，看起来差不多，但真要互通就得写胶水代码。社区里流传一个段子："做一个 Agent 需要两小时，让他能用别人的工具需要两周。"

MCP 1.0 至少推翻了这件事。LangChain 在 7 月 5 日就宣布完全兼容，AWS Bedrock 在 7 月 9 日支持了 MCP。阿里云 7 月 14 日开源了 MCP 适配器，腾讯云 7 月 16 日跟进。到 7 月 18 日，生态里已经有 300 多个插件和连接器。

300 这个数字不算大（Slack 的应用市场有好几千），但 MCP 是个 7 月刚定稿的协议，这个增速已经很快了。

当然也得说句实话：MCP 还没解决 Agent 之间的信任问题。协议定义了怎么通信，但没说"你凭什么相信对方返回的数据"。这后面的坑，等大家真的让 Agent 互相调来调去的时候才会浮出来。

## AutoGen 0.9：Agent 学会"吵架"了

微软的 AutoGen 在 7 月 15 日发布了 0.9 版本，主打一个东西：动态协商机制。

名字听着很学术，实际上做的事挺有意思。多个 Agent 之间可以自动协商谁来做哪个任务。不是以前那种"你写代码、他写测试"的静态分工，而是运行时根据上下文动态分配。

比如你丢一个任务"帮我调研这三家公司的产品定价"，AutoGen 0.9 会让 Agent 们自己商量：谁擅长爬虫、谁擅长数据分析、谁擅长写报告。有冲突（比如两个 Agent 都想做同一件事）就自动协调。

性能提升 40%，这是官方数据。但我更在意的是方向对不对。

多 Agent 协作的瓶颈从来不是性能，而是怎么让它们不互相拆台。我在 Hermes Agent 里做过类似的事情。当子 Agent 并行工作时，每个 Agent 都觉得自己是对的，汇总阶段经常出现自相矛盾的结果。AutoGen 的动态协商机制试图解决这个问题，但说实话，从实验到可用的距离，可能比 0.9 到 1.0 的版本号跨度还要大。

## 企业落地：终于不只是 PPT 了

今年和去年的最大不同是，企业客户开始真金白银地往里砸了。

Accenture 是最激进的。7 月 14 日宣布内部部署了 2500+ AI Agent，覆盖客服、代码审核、采购流程。预计每年节省 4.5 亿美金。这数字我不确定能不能兑现，但敢公开说就是一种态度。

摩根大通比较务实，7 月 12 日上线了 JPM Agent Platform，覆盖合规检查和交易报告这些内部流程。200+ 流程，平均处理时间缩短 60%。60% 这个数字其实比我预期的高。银行合规领域，很多事不是做快就能解决的，还得保证不出错。

Salesforce 的 Einstein Service Agent 在 7 月 5 日全面商用，号称有 10000+ 客户在用，35% 的客服交互由 Agent 处理。这个数据我倾向于相信。Salesforce 已经有现成的客户基础和客服场景，Agent 从 CRM 里的一个功能变成独立产品，路径很短。

西门子做了一件更有意思的事。7 月 19 日推出了 Industrial Agent 套件，用于工厂维护。这跟写字楼里的 Agent 完全是两码事。工厂的传感器数据、PLC 控制、预测性维护，这些东西的规则性很强，数据质量也高（相对互联网数据而言），反而可能是 Agent 最容易落地的场景。

京东的供应链 Agent "京羚"已经处理了 80% 的日常采购询盘。字节的飞书 Agent 企业版拿到了 500 家企业内测。

坦白说，看到 500 这个数字我有点失望。飞书的体量、字节的技术实力，500 家内测说明企业级 Agent 产品还在非常早期的阶段。

## 所以，Agent 真的来了吗？

挺难一刀切的。

基础设施确实在成熟。MCP 解决了 Agent 之间的沟通标准，AutoGen 0.9 让多 Agent 协作更像那么回事，LangChain 0.5 重构了 Actor 模型。这些对开发者是实打实的好处。

但站在用户角度，Agent 产品的体验距离"靠谱"还有一段路。

现在的 Agent 像 2015 年的智能音箱。你知道它以后会很厉害，但现在只能用它做点简单的事。设个提醒、查个天气，可以。让它处理复杂流程、做多步推理，大概率会在某一环跑偏，而且你很难定位问题在哪。

我自己的 Hermes Agent 也是这样。它能帮我做重复性的代码工作、写文章草稿、做信息搜索。但我不敢让它全权代理某个核心工作流。不是能力不行，是我不确定它在边缘情况下的行为。

这可能就是 2026 年 Agent 的现状：基础设施正在就位，资本正在涌入，但真正值得信赖的 Agent 产品，还需要几个迭代。

我没法告诉你"Agent 的 iPhone 时刻"什么时候来。但能确定的是，地基铺得差不多了。
