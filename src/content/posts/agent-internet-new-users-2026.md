最近看到一个数据，让我愣了好一会儿。

Oasis Security 2025 年的报告显示，全球企业中平均每个员工对应 144 个非人类身份。一年前是 92:1。也就是说，你公司系统里跑的 Agent 数量，早就是人类的上百倍了。

我说的不是什么科幻场景。就是现在，2026 年 6 月的事。

## MCP 和 A2A：Agent 需要自己的通信协议

你肯定听过 MCP，Model Context Protocol，Anthropic 推的那个。它让 Agent 能连上各种工具和服务，有点像 USB-C 的角色。现在已经有超过 17,000 个 MCP 服务器在跑了。

但 MCP 有个问题没解决：Agent 和 Agent 之间怎么通信？

所以 Google 在 2025 年 5 月推了 A2A 协议，Agent-to-Agent，Microsoft 随后跟上。LangGraph、CrewAI、Pydantic AI 这些不同框架的 Agent，理论上能通过 A2A 互相协作。2026 年初有人在 Hacker News 上展示了个 A2A 基础设施，专门给 Agent 之间做商业谈判。

我第一次看到的时候，脑子里冒出的念头是：Agent 要开始做生意了？

## 成本和能力的拐点

几个数字挺有意思。

2025 年 AI 风投是 2110 亿美元，占全球 VC 总额一半。推理成本三年降了 92%，从每百万 token 30 美元到 0.10-2.50 美元。Claude Opus 4.6 在 GPQA Diamond 测试拿了 91.3%，人类专家 69.7%。Agent 自主工作时长从 2024 年初的 4 分钟涨到 2026 年 2 月的 14.5 小时。

推理成本降到 0.10 美元每百万 token 的时候，Agent 工作流就从奢侈品变成了基础设施。就像 AWS 让创业公司不用自己买服务器，便宜推理让每个人都能养一群 Agent。

## 互联网正在为 Agent 重建

互联网的商业模式是注意力经济。广告、点击、滚动浏览。但 Agent 不点广告，不被推荐分心，不会因为"相关内容"多停留半分钟。

当 Agent 变成主要的网络内容消费者，注意力经济的底层逻辑就撑不住了。

有个叫 George Larson 的开发者做了个挺酷的项目：7 美元/月的 VPS 上跑了个 AI Agent，用 IRC 当通信层，连上自己的 GitHub 仓库。访客问 Agent 关于他代码的问题，Agent 会克隆仓库、数测试、读 CI 配置，然后给出具体答案。

这个项目在 Hacker News 上拿了 340 个赞。不是因为技术多厉害，是因为它展示了 Agent 可以直接消费代码，不需要人写的简历做中间层。

## 问题在哪

Agent 的错误率是指数级累积的。单步 95% 成功率听起来不错，链 20 步下来，端到端成功率就掉到 36%。这就是为什么生产环境里 Agent 还是做单步任务为主。不是多步编排做不到，是可靠性数学太残酷。

治理也是个问题。欧盟 AI 法案第 50 条 deadline 是 2026 年 8 月 2 日，加州和纽约也快跟上了。多数公司连自己有多少 Agent 在跑都说不清楚。

我挺喜欢 Statewright 这个项目，用可视化状态机让 Agent 行为可预测、可审计。Hacker News 上 126 个赞，说明大家对 Agent 可靠性的焦虑是真的。

## 我怎么看

MCP 和 A2A 不会是最终协议。TCP/IP 也迭代了无数次才稳定，Agent 协议也会继续演化。7 月份 MCP 规范已经有重大变更了。

但方向没问题。80% 的 Neon 数据库是 Agent 创建的，GitHub 上 4% 的 commit 是 Claude Code 写的（预计年底超 20%），Cursor 24 个月从零到 10 亿美元 ARR。这些不是什么边缘现象。

我个人最期待的是 Agent 之间的协作。一个写代码，一个做安全审计，一个负责部署，通过 A2A 协议互相协调。各个组件都已经存在了，只是还没人把它们拼好。

谁先拼好，谁就吃到最大的那块蛋糕。
